# Direccion visual - AVX Mission Control

## Concepto
`monitor` debe sentirse como una cabina de control operativa: precisa, densa, legible y enfocada en senales.

Referencias internas de lenguaje:
- cabina de nave
- radar
- system matrix
- threat console
- telemetry stream

## Principios
- La informacion manda sobre la decoracion.
- Los colores indican estado, no solo estilo.
- Las pantallas deben poder leerse rapido en desktop y mobile.
- Evitar una UI de marketing; esto es una herramienta de operacion.

## Componentes esperados
- Paneles de estado.
- Listas compactas.
- Indicadores con severidad.
- Timeline o stream de eventos.
- Matriz de checks por sitio.

## Color
Usar una base oscura o neutra solo si la legibilidad se mantiene.

Estados:
- OK: verde controlado.
- Advertencia: amarillo/ambar.
- Critico: rojo.
- Info: azul o cyan.
- Deshabilitado: gris.

## Responsive
Desktop:
- Permite densidad alta.
- Dashboard con resumen y matriz.

Mobile:
- Priorizar lista de sitios, severidad y accion principal.
- Evitar tablas anchas.
- Los paneles deben apilarse sin perder contexto.

## Accesibilidad basica
- Contraste suficiente.
- Estados no deben depender solo del color.
- Botones con texto o icono reconocible.
- Foco visible en controles interactivos.
